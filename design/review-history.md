## Iteration v1 — 71/100

### 항목별 점수
- IA: 15/20 · 발견성: 16/20 · 투표 UX: 10/15 · 안전: 11/15 · 엣지: 6/10 · 시스템: 7/10 · 접근성: 6/10

### 강점
1. 주최자/참여자 플로우 분리 명확
2. 핵심 행동(하트/쪽지/투표 배너) 전면 배치
3. 초기 동의 화면에 익명/신고 정책 노출

### Critical 개선 (v2에서 반영)
1. **1·2차 투표 상태 모델 UI화** — 내 선택 n명, 단계 전환 규칙, 선택됨/미선택 배지
2. **하트 오동작 방지** — 카드/버튼 탭 영역 분리, 취소 확인 바텀시트, 마감 직전 경고
3. **안전 맥락 강화** — 상세/쪽지에서 신고 1차 액션, 차단과 신고 구분
4. **상태 디자인 토큰 추가** — disabled/focus/selected/primary-soft/warning-surface/overlay + 컴포넌트 상태 규격

### 보조 개선
- 약관 체크박스 라벨 링크화
- 검색 결과 없음 / 로딩 skeleton / 전송 실패 재시도
- 아이콘 단독 버튼 aria-label, 색에 의존하지 않는 상태 표시
- 주최자 방 상세: 현재 단계에 따라 primary 액션 1개만, 위험 액션은 danger 영역

## Iteration v2 — 83/100

### 항목별 점수
- IA: 17/20 · 발견성: 17/20 · 투표 UX: 13/15 · 안전: 13/15 · 엣지: 7/10 · 시스템: 8/10 · 접근성: 8/10

### 개선된 부분
1. 1·2차 투표 상태 모델이 실제 UI에 드러남 (내 선택 n명, 타임라인, 1차 선택됨 배지)
2. 하트 탭존 분리 + 취소 바텀시트 + 마감 잔여시간 → 오동작 리스크 감소
3. 신고 상단 1차 액션 + 신고 vs 차단 구분 카드
4. 상태 토큰(disabled/focus/selected/overlay) + 컴포넌트 상태 규격 추가

### Critical (v3 반영)
1. **2차 투표용 선택 검토/수정 전용 화면** — 유지/신규/해제 예정 상태를 빠르게 확인·수정
2. **신고/차단 맥락 분리** — 프로필 신고 vs 쪽지 신고 / 차단 후 결과 화면
3. **DS 확장** — 입력/체크박스/탭/카드/카운트다운 + 실패 상태 규격
4. **접근성 인터랙션 규격** — 카운트다운 읽기 빈도, live region 문구, 키보드 포커스 시나리오

### 보조
- 홈 CTA 라벨 분리 ([선택 검토] vs [참여자 보러가기])
- 주최자 조기 마감 영향 범위 명시 ("즉시 투표 종료, N명 영향, 되돌릴 수 없음")
- 홈/상세/모달 계열 실패 상태 (신고 제출 실패, QR 생성 실패, 네트워크 재연결)
