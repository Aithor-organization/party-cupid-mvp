# Wireframes v3 — Day 3 Critical Path + 선택 검토 화면

> **v2 → v3 변경**:
> 1. **A-9 /my-hearts 신설** — 2차 투표 선택 검토/수정 전용 화면
> 2. 홈 CTA 분리 (`[선택 검토]` vs `[참여자 보러가기]`)
> 3. 신고 맥락 분리 — 프로필 신고 vs 쪽지 신고, 차단 후 결과 화면 추가
> 4. 주최자 조기 마감 영향 범위 명시 + 2단계 확인 와이어
> 5. 실패 상태 구체화 (QR 생성 실패, 신고 제출 실패, 네트워크 재연결)
> 6. 접근성: aria-live 문구, 카운트다운 낭독 규칙 (design-system.md 참조)

```
│ ┌──────────────────┐  │
│ │ ◎ 02 라이트        │  │
│ │   1차에 선택 → 2차  │  │
│ │   에서도 유지 중    │  │
│ │              [❤️]  │  │  ← 클릭 → 해제 예정으로
│ └──────────────────┘  │
│ ┌──────────────────┐  │
│ │ ◎ 05 달빛          │  │
│ │   1차에 선택 → 유지 │  │
│ │              [❤️]  │  │
│ └──────────────────┘  │
│                        │
│ ─── ✨ 신규 (1명) ─── │
│ ┌──────────────────┐  │
│ │ ◎ 08 오렌지빛      │  │
│ │   2차에서 새로      │  │
│ │   선택함            │  │
│ │              [❤️]  │  │
│ └──────────────────┘  │
│                        │
│ ─── 🗑 해제 예정 (0) ─ │  (빈 상태)
│   (아직 해제한 사람이   │
│    없어요)              │
│                        │
│ ─── 💡 안내 ───        │
│ 2차 마감 시점 기준으로  │
│ 선택한 사람과 양방향     │

│                        │
│ [참여자 더 추가하기 →] │
│                        │
└───────────────────────┘
```

**v3 핵심**:
- 각 참여자 카드에 상태 배지 (유지 / 신규 / 해제 예정) 명시
- 하트 토글: 유지/신규 → 해제 예정, 해제 예정 → 유지/신규로 복구
- 필터 탭으로 상태별 빠른 확인
- 마감 직전: 상단 sticky 경고 "⏰ 마감 3분 전, 최종 확인!"

### 해제 예정 카드 스타일
```
┌──────────────────┐
│ ◎ 02 라이트        │  ← border dashed warning
│   ⚠ 해제 예정      │
│   마감 시점에 취소   │
│              [♡]  │  ← 빈 하트, 다시 누르면 유지로 복구
└──────────────────┘
```

## 변경 요약 (v2 → v3)

| 항목 | v2 | v3 |
|------|-----|-----|
| 선택 검토 UI | 홈 카드 요약만 | **A-9 전용 페이지 추가** |
| 상태 배지 | 1차 선택됨만 | 유지 / 신규 / 해제 예정 3종 |
| CTA | "내 선택 보기" (모호) | "선택 검토" + "참여자 보기" 분리 |
| 신고 | 상세 상단 1개 | 프로필 신고 + 쪽지 신고 분리, 옵션 메뉴로 정리 |
| 차단 후 | 미정의 | 차단 결과 화면 + 해제 경로 |
