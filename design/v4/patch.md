## Patch 1 — A-7 참여자 목록 (라운드 자동 전환 + 필터 재구성)

### ROUND_BANNER 컴포넌트 (신규 규격)

라운드별 경고/안내가 **현재 단계**에 따라 자동 치환:

| 상태 | 배경 | 아이콘 | 텍스트 | 위치 |
|------|------|--------|--------|------|
| 1차 투표 중 (일반) | accent-soft | 🌸 | "1차 투표 진행 중 · 첫인상으로 선택" | 상단 sticky |
| 1차 마감 1분 전 | warning-surface | ⏰ | "1차 마감 1분 전 · 지금 선택이 최종" | 하단 sticky |
| 1차↔2차 사이 | bg | 💬 | "대화 시간 · 22:00에 2차 시작" | 상단 |
| 2차 투표 중 (일반) | accent-soft | 💜 | "2차 투표 진행 중 · 1차 선택 유지/변경 가능" | 상단 sticky |
| 2차 마감 1분 전 | danger-soft | ⏰ | "2차 마감 1분 전 · 이번이 최종 선택" | 하단 sticky |
| 투표 종료 | disabled-bg | 🔒 | "투표 종료 · 매칭 결과 확인" + [결과 보기] | 상단 |

**규칙**:
- 클라이언트 타이머가 단계 전환 시 컴포넌트 prop만 교체 (문구 수동 변경 금지 — 버그 원인)
- `aria-live="polite"` → 전환 시 스크린리더 알림
- 마감 1분 전 경고는 `aria-live="assertive"`

### A-7 필터 탭 — 라운드별 자동 재구성

## Patch 2 — 하트 저장 실패 / 동시성 / 저장 중 상태 (신규)

### HEART_SAVE_STATE 컴포넌트 (신규)

하트 버튼에 **저장 상태 인디케이터** 추가:

| 상태 | 시각화 | 텍스트 보조 |
|------|--------|-----------|
| `idle` | 기본 하트 | - |
| `saving` | 하트 + 작은 spinner 우상단 | "저장 중..." (aria-live) |
| `saved` | 하트 + ✓ 페이드 (0.8s) | "저장됨" |
| `save-failed` | 하트 + ⚠ 빨간 점 | "저장 실패 · 탭하여 재시도" |
| `conflict` | 하트 + 🔄 주황 | "다른 기기에서 변경됨 · 새로고침" |
| `closing-saving` | 하트 + ⏳ | "마감 직전 저장 중..." |

### 하트 저장 실패 인라인 배너 (리스트/상세 공통)

```
┌────────────────────────────────┐
│ ⚠ 하트 저장 실패 2건            │
│                                │
│ • 02 라이트 · 저장 실패          │
│ • 08 오렌지빛 · 동시성 충돌      │
│                                │
│ [모두 재시도] [하나씩 보기]     │
└────────────────────────────────┘
```

### 재시도 정책

- 자동 재시도: 2초/4초/8초 (exponential backoff, 최대 3회)
- 3회 실패 시 수동 재시도 버튼 노출
- 동시성 충돌(409): 자동 새로고침 후 최신 상태 반영 → 사용자 확인 요청

### 마감 직전 저장 상태 (A-7, A-8, A-9 공통)

마감 10초 전 모든 변경 요청:
```
┌───────────────────────┐
│  ⏳ 저장 중...          │
│  마감 직전 모든 변경     │
│  을 저장하고 있어요      │
└───────────────────────┘
```

마감 시점 최종 저장 완료 토스트:
```
┌─────────────────────────────┐
│ ✓ 최종 선택이 저장됐어요     │
│  (22:30:00 기준)             │
└─────────────────────────────┘
```

### 최종 저장 시각 표기 (A-9)

```
┌───────────────────────┐
│ ← 내 선택 검토          │
│ 💜 2차 투표 진행 중      │
│ 마감 00:22:15            │
│ 🕐 마지막 저장 22:07:48  │  ← v4 신규
└───────────────────────┘
```

## Patch 3 — A-9 상태 복구 피드백 강화

### 해제 예정 → 복구 규칙 (명시)

- **원래 유지였던 것**: 해제 예정 → **유지**로 복구
- **원래 신규였던 것**: 해제 예정 → **신규**로 복구
- 규칙은 카드 내 항상 표시 + 복구 시 토스트로 재확인

### 카드 상태 라벨 (v4 강화)

```
[유지 → 해제 예정 상태]
┌──────────────────────┐
│ ◎ 02 라이트            │
│ ✓ 유지 → 🗑 해제 예정   │  ← 전환 히스토리 표시
│   원래 상태: 유지        │  ← 복구 힌트
│   다시 누르면: 유지 복원 │
│                  [♡]  │
└──────────────────────┘
```

### 상태 전환 토스트 (v4 신규)

| 액션 | 토스트 문구 | 역할 |
|------|-----------|------|
| 1차 하트 누름 | "❤ 1차 선택 (신규)" | success |
| 2차에서 유지 상태의 하트 취소 | "🗑 해제 예정으로 전환 · 마감 시 취소됨" | warning |
| 해제 예정 상태 복구 | "✓ 유지로 복원됨 (원래 상태)" | success |
| 2차에서 새 사람 하트 | "✨ 신규 선택" | success |
| 신규 → 해제 | "🗑 신규 선택 취소됨" | warning |

### 카드 내 즉시 피드백

하트 토글 직후 0.3s 애니메이션:
- unselected → selected: 하트 scale 1.2 → 1.0, 배경 fade-in primary-soft
- selected → 해제 예정: border dashed warning 점선 애니메이션
- 해제 예정 → 복구: border dashed → solid, 상태 라벨 페이드 전환

`prefers-reduced-motion` 시 즉시 상태 변경 (애니메이션 생략)

## Patch 5 — 접근성 포커스 관리 보강

### FOCUS_MANAGEMENT 규칙 (v4 추가 — design-system.md에 병합 권장)

#### 1. 바텀시트 / 모달

- 열림 시: 첫 번째 버튼 또는 첫 입력 필드에 `autofocus`
- 닫힘 시: **원래 트리거 버튼으로 포커스 복귀**
- Tab 순환: focus trap (마지막에서 Tab → 첫 요소)
- ESC: 닫기 + 트리거 포커스 복귀

#### 2. 탭 전환 (A-7 필터, H-3 탭)

- 필터 탭 클릭 시: 결과 리스트 첫 아이템에 포커스 이동
- `aria-live="polite"`: "2차 필터 유지 2명 표시" 안내
- 키보드: ← → 로 탭 이동, Enter/Space로 활성화
- 활성 탭 `aria-selected="true"` + `tabindex="0"`, 비활성 `tabindex="-1"`

#### 3. 차단 후 결과 화면

- 화면 진입 시: 첫 CTA `[차단 해제하기]`에 포커스
- 스크린리더 announcement: "라이트 #02님을 차단했습니다. 설정에서 해제할 수 있어요"
- 흐름 복구 액션 (v4 신규):
  ```
  ─── 다음으로 ───
  [참여자 목록으로]
  [쪽지함으로]
  [홈으로]
  ```

#### 4. 신고 제출 완료

- 바텀시트 내부에서 성공 메시지 표시 (화면 전환 없음)
- 포커스: "닫기" 버튼으로
- 스크린리더: "신고가 접수되었습니다. 주최자가 확인할 거예요"

#### 5. 하트 토글

- 포커스: 버튼에 유지 (이동 없음)
- `aria-pressed` 상태 변경
- 토스트가 `aria-live="polite"`로 저장 결과 낭독

#### 6. 투표 단계 전환 (시스템 자동)

- 현재 포커스 위치 유지 (강제 이동 금지)
- `aria-live="polite"` 배너로 안내
- 5초 후 silent mode (연속 낭독 방지)
